01   PROBLEM & SOLUTION
PROBLEM

Claude Code developers write valuable lessons into docs/skills.md files — but those entries stay siloed per project. When starting a new project or hitting a problem you have solved before, there is no way to search across all your accumulated knowledge. Lessons written in the RMA project are invisible when working on the CRM project.

SOLUTION

A SessionStart hook automatically launches a local web server on port 38888. On startup, it parses ~/.claude/global-skills.md into a SQLite FTS5 index. A PostToolUse hook syncs new entries from any project into that global file. The viewer is already running before the first keystroke of the session — zero friction.

02   ARCHITECTURE
Claude Code session starts │ ▼ SessionStart hook → start-viewer.ps1 │ ├── Port 38888 occupied + HTTP probe passes → skip (already running) │ └── node viewer/server.js │ ├── parseMd() → ~/.claude/global-skills.md → SQLite FTS5 ├── fs.watch → auto-resync on file change (500ms debounce) ├── PID file → ~/.claude/global-skills.pid └── :38888 ├── GET / → viewer/public/index.html ├── GET /api/skills → FTS5 search + LIKE fallback └── POST /api/sync → manual re-parse On any Write to docs/skills.md: PostToolUse hook → sync-skills.ps1 → append to global-skills.md → fs.watch picks up change → DB resyncs

05   KEY DECISIONS
D1
Flat MD as canonical source, SQLite as cache — not the other way around
The MD file is readable without the app. The DB is always rebuildable. Source of truth is never a binary file.
D2
Zero-dependency viewer (no React, no bundler)
A 3-column dark UI with search and FTS5 requires no framework. Vanilla JS + fetch + CSS custom properties. Ships as a single HTML file.
D3
Fixed port 38888 — no dynamic port negotiation
Single-user dev machine. A fixed port means a fixed URL you can bookmark. Dynamic ports would require a port discovery step on every session.
D4
HTTP probe before skipping start — not just netstat
netstat shows TIME_WAIT connections that look occupied but aren't. Probing /api/skills confirms it's actually our server, not a leftover socket.
D5
Ship v1.1.0 first, in-session recommendations in v1.2
The viewer is the wedge — it proves value before investing in the harder PreToolUse context-injection feature. Get community usage data before building deeper.
06   WHAT'S NEXT
v1.2.0 — IN-SESSION RECOMMENDATIONS

A PreToolUse hook queries global-skills.db for entries matching the current task context and injects the top 1–2 results into Claude's context automatically. The viewer becomes optional — relevant knowledge surfaces while you work.

v2.0.0 — TEAM SYNC

Replace the flat MD file with a shared git repo or hosted API backend. Multiple developers push and pull skill entries. The knowledge base survives employee turnover and onboarding gaps. This is the version with network effects.
